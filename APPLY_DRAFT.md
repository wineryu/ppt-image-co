# 小米百万亿 token 计划申请草稿

## 04 项目成果描述

我正在使用 Codex 构建一个视觉优先的 PPT 分镜工具 `PPT Image First`。项目目标是解决传统 AI PPT 生成中过度依赖文字大纲、视觉表达弱、页面风格不一致的问题。这个工具会先围绕演示主题生成每页的视觉锚点，再补充页面文案和图片提示词，让演示稿从一开始就围绕画面叙事组织。

目前的核心流程是：用户输入主题、听众、页数和视觉风格；工具生成每页的页面目标、视觉锚点、短文案和图像提示词；随后可以导出 Markdown 或 JSON，用于继续交给 Agent 迭代、生成图片、改写文案或接入 PPTX 导出。Codex 在这个项目中参与了需求拆解、页面设计、前端实现、交互逻辑、导出格式和申请材料整理。

这个项目需要较大的 token 额度，因为真实的演示稿创作不是一次生成就完成，而是需要反复参考代码、页面截图、视觉目标、提示词、导出文档和用户反馈进行多轮修改。更高额度可以支持我持续扩展模板库、接入真实模型调用、做跨文件重构、进行截图审查，并把这个工具迭代成完整的 Agent 辅助 PPT 创作流程。

## 简短版

我用 Codex 构建了一个视觉优先的 PPT 分镜工具。它可以把一个演示主题拆成页面目标、视觉锚点、短文案和图片提示词，并导出 Markdown/JSON 供后续 Agent 继续迭代。这个项目解决 AI PPT 生成容易文字堆叠、视觉不一致的问题。高 token 额度能支持我进行长上下文、多文件、多轮视觉迭代，把它扩展为可用的 Agent 辅助 PPT 创作工作流。
